Configuring NGINX on AWS: A DevOps Case Study

By DJIOTSA DJOUAKE CHRISTIAN DARYN | LinkedIn Profile

Abstract

This case study documents the process of configuring an NGINX web server on an Ubuntu instance hosted on AWS. The objective was to deploy a functional web server capable of serving a custom HTML page. The exercise highlights key DevOps practices, including infrastructure provisioning, server configuration, and deployment automation.

Methodology

The task was executed in the following phases:

  1. Infrastructure Provisioning: An Ubuntu 22.04 LTS instance was launched on AWS EC2. The security group was configured to allow HTTP (port 80) and SSH (port 22) traffic.
  2. Server Configuration: The instance was accessed via SSH using a private key (.pem file). NGINX was installed using the apt package manager, and the service was started and enabled for automatic startup on boot.
  3. Custom HTML Deployment: The default NGINX HTML file (/var/www/html/index.html) was replaced with a custom page displaying the message: "Welcome to DevOps Stage 0 - [Your Name]/[SlackName]".
  4. Validation: The web server was tested by accessing the instance’s public IP address in a browser. The custom HTML page was successfully served, confirming the configuration’s correctness.

Challenges and Solutions

Several challenges were encountered during the process:

  • AWS EC2 Configuration: Initial unfamiliarity with AWS EC2 was addressed by referencing the official AWS documentation and tutorials.
  • SSH Connectivity: Permission issues with the private key were resolved by setting the correct file permissions using chmod 400 <key-file>.pem.
  • NGINX File Placement: Uncertainty about the default HTML file location was resolved by consulting the NGINX documentation, which indicated /var/www/html/index.html as the default path.

Conclusion

This exercise demonstrated the foundational skills required for DevOps, including cloud infrastructure management, server configuration, and deployment automation. The successful deployment of a custom HTML page using NGINX on AWS EC2 underscores the importance of understanding both theoretical concepts and practical implementation in DevOps.

Future work will focus on advanced topics such as infrastructure as code (IaC) using Terraform, continuous integration/continuous deployment (CI/CD) pipelines, and containerization with Docker and Kubernetes.
